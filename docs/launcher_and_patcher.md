# Launcher & Update System

## silkroad.exe - Launcher

**Size**: 760 KB
**Type**: MFC (Microsoft Foundation Classes) application

### Purpose

The launcher is the primary entry point for the game. It handles:
- Version checking against the server
- Patch detection and downloading
- Server selection
- Launching the game client (`sro_client.exe`)

### Server Connection

The launcher connects to the same server infrastructure as the game client, using a similar network stack:

- **GatewayServer** - Initial connection for version check and server list
- **AgentServer** - Referenced but likely used for game launch coordination
- **DownloadServer** - Dedicated file download server for patches

The launcher includes its own handshake implementation:
```
ACTIVE_SESSION::_OnMsgReceivedBeforeHandshake() - Handshake ServerSignature Error[MsgID: 0x%x]
```

### Configuration Files

| File | Purpose |
|------|---------|
| `GATEPORT.TXT` | Gateway server IP/port configuration |
| `launcher\reflinkurl.txt` | URL references for launcher resources |
| `launcher\patch.dat` | Patch status/version data |

### UI Resources

The launcher includes embedded UI assets:
- `launcher\update_focus.dat` - Update button focus state
- `launcher\update_select.dat` - Update button selected state
- `launcher\window_update.dat` - Update window layout

### DLL Dependencies

Core system DLLs plus:
- `GFXFileManager.dll` - PK2 archive access
- `WS2_32.dll` - Network communication
- `WINMM.dll` - Multimedia
- `comctl32.dll`, `comdlg32.dll` - Common controls/dialogs
- `OLEACC.dll`, `OLEAUT32.dll` - OLE/Accessibility
- `SHLWAPI.dll` - Shell utility functions
- `dbghelp.dll` - Crash dump support
- DirectX DLLs referenced: d3d8.dll, d3d9.dll, d3drg8x.dll, ddraw.dll, dinput.dll, dplayx.dll, dpnet.dll

### Duplicate Launch Prevention

```
Silkroad is already executed!!
```

The launcher checks for an existing running instance and prevents duplicate launches.

### Shutdown Message

```
Now Silkroad Online Launcher is closing..
```

## replacer.exe - Updater

**Size**: 320 KB
**Description**: "Silkroad Online Updater"
**Type**: MFC application (`CReplacerApp` / `CReplacerDlg`)

### Purpose

Handles post-download file replacement during the patching process. This is a separate executable because it needs to replace files that may be locked by the main launcher.

### Update Process

1. Reads update packages from `%s\%s\Replace.pk` path pattern
2. Replaces game files with updated versions
3. Verifies `Media.pk2` integrity after patching
4. Writes updated version to registry

### Media.pk2 Integrity Check

```
Media.pk2 file is damaged. Can't save version.
```

After patching, replacer.exe validates the Media.pk2 archive. If corrupted, it refuses to update the version number, forcing a re-download on next launch.

### Registry Keys

- **`SILKROADVERSION`** - Stores the current client version number (written after successful patch)

### Error Messages

| Message | Cause |
|---------|-------|
| `Failed to replace %s file. Please Restart Your System and Try agin..` | File locked or permission denied (note: typo "agin" is in the original) |
| `Update failed with error_code[%d]` | General update failure |
| `Update temp-file [%s] is damaged. Please try again.` | Downloaded patch file corrupted |
| `Failed to write updated version.` | Cannot write version to registry |

## silkerrsender.exe - Crash Reporter

**Size**: 252 KB
**Type**: MFC application (`CFTPSenderApp` / `CFTPSenderDlg`)

### Purpose

Collects and sends crash dump files (`.dmp`) to JoyMax's crash analysis servers via FTP.

### Implementation

- Uses `CFtpConnection` and `CInternetConnection` (MFC Internet classes)
- Calls `FtpPutFileA` to upload dump files
- Connects via `InternetConnectA`

### Crash Dump Collection

The sender looks for `.dmp` files generated by the client's `MiniDumpWriteDump` call. Internal string references:
- `"joymaxdump"` - dump identifier/prefix
- `"memory dump"`, `"arg buff memory dump"`, `"original memory dump"` - dump type descriptors

### Server-Dependent Resources

```
server_dep\%s\textdata\textuisystem.txt
```

The crash sender loads localized UI text from a server-dependent path, supporting multiple language deployments.

### UI Strings

| String ID | Purpose |
|-----------|---------|
| `UIIS_STT_MSG_SEND_ERROR` | Send error notification |
| `UIIS_STT_MSG_SEND_ERROR_PLZ` | Send error with retry prompt |
| `UIIS_STT_QUESTION_MSG_SEND_ERROR` | Send error confirmation dialog |

### Upload Target

The crash dump upload destination is **`dmp.sro.joymax.com`**, passed as a command-line argument:

```
"%s" /"dmp.sro.joymax.com" "%s" "%d" "%d" "%d"
```

The game client (`sro_client.exe`) launches `silkerrsender.exe` with this command line, passing the FTP host, dump file path, and numeric identifiers.
